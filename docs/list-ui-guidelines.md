# دليل شكل القوائم (List UI Guidelines)

هذا الملف مرجع عملي موحّد لشكل القوائم في النظام حتى نستكمل التطوير بنفس النمط.

## 1) النمط العام للقائمة

- صفحة القائمة تستخدم `PagedResult<T>` مع عرض بيانات الصفحة الحالية + إجماليات.
- البحث يدعم: عام + أعمدة + نمط (`contains / starts / ends`).
- أحجام الصفحات المعتمدة: `10, 25, 50, 100, 200` و`0` = الكل.
- أزرار الترتيب تكون بنفس نمط القوائم الحالية (asc/desc مع حفظ الحالة في `ViewBag`).

## 2) سلوك الفلاتر

- فلاتر الأعمدة بنمط Excel (قائمة قيم + بحث داخل القيم).
- دعم التعبيرات الرقمية في الأعمدة (`<`, `>`, `x:y`) عند الحاجة.
- عند وجود أكثر من نموذج يرسل نفس الحقول في query يجب التطبيع قبل التطبيق.

## 3) الأداء في القوائم

- الاستعلام الأساسي يكون `AsNoTracking()` في صفحات القراءة.
- يتم حساب `Count/Sums` على نفس `query` بعد تطبيق الفلاتر وقبل `Skip/Take`.
- أي صلاحيات عامة مطلوبة للقوائم (`Global.*`) تعتمد على كاش الصلاحيات.

## 4) واجهة المستخدم

- الجداول القياسية تستخدم سكربتات:
  - `wwwroot/js/erp-list-column-filter.js`
  - `wwwroot/js/erp-list-selectable.js`
  - `wwwroot/js/erp-list-print.js`
- كروت الإجماليات في القوائم تكون موحّدة: Label أعلى + Value أسفل.
- الحفاظ على نفس تسميات الحقول داخل كل موديول (لا تغيير مصطلحات بدون تحديث شامل).

## 5) Checklist عند تعديل أي قائمة

1. توحيد `pageSize` وقيمه المعتمدة.
2. الحفاظ على `searchMode` والتطبيع.
3. التأكد من تطبيق صلاحيات العرض للقائمة.
4. اختبار الأداء على بيانات فعلية (Count + Sum + Page query).
5. تحديث هذا الملف أو `DEVELOPER_GUIDE.md` عند أي قاعدة جديدة.

## 6) آخر التحديثات المنفذة (2026-04-21)

### توحيد البحث والفلاتر

- توحيد ترتيب عناصر البحث في القوائم ليبدأ بحقل النص أولاً بنفس نمط قائمة المبيعات.
- توحيد سلوك البحث في الأعمدة الرقمية في قوائم الأصناف (خصوصًا أصناف أوامر البيع) ليطابق قائمة أصناف المبيعات.

### توحيد شكل القوائم (Header + Rows)

- تطبيق نمط القوائم الموحد (هيدر وسطور) على قوائم:
1. قائمة مرتجعات البيع.
2. قائمة أصناف مرتجعات البيع.
3. قائمة أوامر المبيعات.
4. قائمة أصناف أوامر البيع.
5. قائمة طلبات الشراء.
6. قائمة المشتريات.
7. قائمة مرتجعات الشراء.
8. قائمة أصناف المشتريات.
9. قائمة أصناف طلبات الشراء.
10. قائمة أصناف مرتجعات المشتريات.

### توحيد ترتيب الأزرار في القوائم

- ضبط ترتيب أزرار العمليات في قوائم المشتريات/طلبات الشراء/أصناف المشتريات/أصناف طلبات الشراء/أصناف مرتجعات المشتريات ليطابق قائمة المبيعات.
- إزالة زر `حذف` من قوائم السطور حسب الطلب.

### تحسين حقول الإدخال (Saved info)

- إزالة ظهور `Saved info` من بوكسات الأسماء/الأصناف في:
1. طلب الشراء.
2. فاتورة المشتريات.
