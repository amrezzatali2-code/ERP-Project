# رصيد أول المدة — الطريقة المناسبة

الهدف: أي شركة جديدة ندخلها نستورد لها **رصيد أول المدة** (أصناف + عملاء + خزينة) والكود **يحسب** الأرصدة من هذه البيانات مع الحركات اللاحقة.

---

## 1. المبدأ العام

لا ننشئ جداول "أرصدة مستوردة" منفصلة يُعاد حسابها يدوياً. نستخدم **نفس الجداول والمنطق** الذي ي already يحسب الأرصدة، وندخل رصيد أول المدة كـ **حركات/قيود افتتاحية** فيها:

- **الأصناف:** حركات افتتاحية في **StockLedger**
- **العملاء:** قيود افتتاحية في **LedgerEntries**
- **الخزينة:** قيد افتتاحي في **LedgerEntries** على حساب الخزينة

بهذا الشكل:
- تقرير أرصدة الأصناف = رصيد افتتاحي + مشتريات + تسويات + تحويلات - مبيعات - مرتجعات ...
- رصيد العميل = قيود افتتاحية + فواتير + إشعارات - قبض ...
- رصيد الخزينة = قيد افتتاحي + إذون استلام - إذون دفع ...

---

## 2. أرصدة أصناف (كميات + خصم مرجّح + إجمالي تكلفة)

### الجدول: **StockLedger**

- إدخال **حركة افتتاحية** لكل صنف/مخزن (أو لكل صنف/مخزن/تشغيلة إن لزم):
  - **SourceType** = `"Opening"` (نوع ثابت نضيفه إن لم يكن موجوداً)
  - **SourceId** = 0 أو رقم دفعة استيراد (مثلاً 0 لرصيد أول المدة)
  - **SourceLine** = 0 أو رقم تسلسلي
  - **WarehouseId** = مخزن الدواء أو الإكسسوار
  - **ProdId** = الصنف
  - **QtyIn** = الكمية الافتتاحية
  - **QtyOut** = 0
  - **UnitCost** = تكلفة الوحدة (أو إجمالي التكلفة ÷ الكمية)
  - **RemainingQty** = نفس الكمية (لأنها كلها متبقية)
  - **PurchaseDiscount** = الخصم المرجّح % (إن وُجد في الإكسل)
  - **TranDate** = تاريخ أول المدة
  - **BatchNo** / **Expiry** = إن كان الاستيراد حسب تشغيلة

- **الكود يحسب الرصيد:**  
  تقرير أرصدة الأصناف ومزامنة **Stock_Batches** يجمعان من **StockLedger** بدون استثناء نوع الحركة (أو نوسّع الاستعلام ليشمل `SourceType == "Opening"` في جزء الخصم والتكلفة إن كان حالياً مقصوراً على `"Purchase"`).

- بعد استيراد حركات الافتتاح يُفضّل تشغيل **مزامنة الأرصدة** (SyncStockBatchesFromLedger) لضبط **Stock_Batches**.

---

## 3. حسابات العملاء (رصيد أول المدة)

### الجدول: **LedgerEntries**

- **LedgerSourceType.Opening** موجود مسبقاً في النظام ("رصيد افتتاحي").
- لكل عميل له رصيد في الإكسل:
  - قيد (سطر أو سطرين) في **LedgerEntries**:
    - **SourceType** = `Opening`
    - **SourceId** = 0 أو رقم دفعة الاستيراد
    - **LineNo** = 1, 2, ...
    - **AccountId** = حساب ذمم العملاء (أو حساب العميل إن كان لكل عميل حساب)
    - **CustomerId** = العميل
    - **Debit** / **Credit**: المدين = رصيد مدين، الدائن = رصيد دائن (حسب اصطلاحك)
    - **EntryDate** = تاريخ أول المدة

- **الكود يحسب الرصيد:**  
  رصيد العميل = مجموع (مدين - دائن) لقيود هذا العميل في **LedgerEntries** (بما فيها الافتتاحية). لا حاجة لجدول إضافي.

---

## 4. رصيد الخزينة

### الجدول: **LedgerEntries**

- قيد افتتاحي واحد (أو أكثر إن تعددت الخزائن):
  - **SourceType** = `Opening`
  - **SourceId** = 0
  - **AccountId** = **رقم حساب الخزينة** (الذي سترسله كـ "رقم الخزينة")
  - **Debit** = رصيد الخزينة الافتتاحي (إن كان الأصل مديناً)
  - **Credit** = 0
  - **EntryDate** = تاريخ أول المدة

- **الكود يحسب الرصيد:**  
  رصيد أي حساب (ومنه الخزينة) = مجموع قيود هذا الحساب في **LedgerEntries**، فالرصيد الافتتاحي يُحسب تلقائياً مع الحركات اللاحقة.

---

## 5. ملخص سير التنفيذ

| البند              | جدول التخزين   | نوع السجل        | كيف يُحسب الرصيد لاحقاً                    |
|--------------------|----------------|------------------|--------------------------------------------|
| أرصدة أصناف       | StockLedger    | SourceType = "Opening" | Sum(QtyIn - QtyOut) + مزامنة Stock_Batches |
| حسابات العملاء    | LedgerEntries  | SourceType = Opening   | Sum(Debit - Credit) للعميل                 |
| رصيد الخزينة      | LedgerEntries  | SourceType = Opening   | Sum(Debit - Credit) لحساب الخزينة         |

---

## 6. ما يلزم في الكود

1. **StockLedger:**  
   - إدخال حركات الافتتاح بـ **SourceType = "Opening"** (قيمة نصية ثابتة).
   - **تقرير أرصدة الأصناف** حالياً يحسب الخصم المرجّح والتكلفة من حركات **SourceType == "Purchase"** فقط؛ يجب توسيع الاستعلام ليشمل أيضاً **"Opening"** حتى يُحسب رصيد أول المدة في الخصم والتكلفة المعروضين.
2. **استيراد من إكسل:**  
   - أصناف → **Products** + ربط بالمخزن حسب نوع الاستيراد (دواء/إكسسوار).  
   - أرصدة أصناف → إدخال صفوف في **StockLedger** بنوع `"Opening"` (كمية، تكلفة وحدة، خصم مرجّح، مخزن، صنف).  
   - عملاء → **Customers** إن لزم.  
   - أرصدة عملاء → إدخال قيود **LedgerEntries** بـ SourceType = Opening و CustomerId و Debit/Credit.  
   - رصيد الخزينة → قيد **LedgerEntries** بـ SourceType = Opening و AccountId = رقم الخزينة.
3. **بعد استيراد أرصدة الأصناف:**  
   - تشغيل **مزامنة الأرصدة** (SyncStockBatchesFromLedger) لضبط **Stock_Batches**.

بهذه الطريقة يكون **رصيد أول المدة** جزءاً من نفس مصدر الحقيقة الذي يحسب منه النظام الأرصدة، وأي شركة جديدة نستورد لها البيانات ثم **نكمل عليها** بالحركات العادية.
